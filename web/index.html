❓ Comment marche NOVA WALL

🎥 Caméras vs terminaux gérés. Les cases caméra sont tes sessions existantes (Terminal.app) — lecture seule (on ne peut pas taper dedans sans voler ton focus). Les cases géré sont lancées par le mur (tmux caché) — tu écris dedans, tu changes le modèle, tu boucles. Pour contrôler un projet : « ＋ gérer » sur sa caméra, ou ＋ Terminal.
✍️ Écrire dans un terminal. Sur une case gérée, tape en bas dans « écrire… » puis Entrée. Ou clic droit (2 doigts sur le trackpad) sur la case → elle s'agrandit et tu écris un vrai message dedans.
🔗 Liaisons (flèches n8n). Clique 🔗 Liaisons, puis la case source, puis la case cible, puis choisis :
· → Injecter : pousse le contexte de A dans B (1 sens).
· ⇄ Bridge : relais bidirectionnel A↔B.
· 🎓 Apprendre : A et B s'échangent leurs leçons/techniques.
Ensuite clique sur la flèche pour exécuter le transfert (ou la supprimer).
⧉ Fusion. Clique ⧉ Fusion, sélectionne 2 sessions : le mur crée un nouveau terminal enfant (Opus+dnGrok) et lui injecte les 2 contextes — il hérite des technos des deux. Prouvé : ton terminal de fusion a lu et synthétisé les 2 contextes tout seul.
🔁 Loop · 🔄 Modèle. 🔁 sur une case gérée = ré-injecte une instruction à intervalle (ex. « continue » toutes les 2 min). Le sélecteur de modèle (cases gérées) relance vraiment le terminal dans le modèle choisi.
