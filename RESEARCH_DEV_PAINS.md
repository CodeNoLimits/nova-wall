# RESEARCH — Douleurs des devs qui font tourner des agents IA en parallèle
> Recherche web du 2026-07-20 (session 94b2fbde / nova-wall-v2). Sources : GitHub issues `anthropics/claude-code`, Hacker News, essais de devs (Substack/Medium/DEV.to), docs Anthropic, écosystème d'outils (stars/apps), études arXiv.
> **Méthode & limites (honnêteté)** : le classement « fréquence » est basé sur le VOLUME de preuves indexées (nb d'issues GitHub sur le thème, taille de l'écosystème d'outils nés de la douleur, proéminence des threads) — pas un comptage scientifique. Les compteurs de réactions GitHub étaient indisponibles au fetch (interface « Reactions are currently unavailable ») sauf mention contraire. Reddit est mal indexé par le moteur de recherche utilisé : les douleurs Reddit sont captées ici via les articles/outils qui en sont nés et les issues GitHub qui les citent.

## 1) TOP 10 DES DOULEURS (classées par volume de preuves)

### 🥇 P1 — Perte de contexte / auto-compact = « Claude a tout oublié »
La douleur la plus documentée du corpus : la compaction automatique du contexte détruit l'état de travail en pleine session.
- **Citation exacte** (issue [#13112](https://github.com/anthropics/claude-code/issues/13112), *closed as not planned*) : « *auto compact is the worst. every time it happens i feel like claude code has forgotten everything — whats the point of the auto-compact if it functionally makes everything worse?* »
- [#868](https://github.com/anthropics/claude-code/issues/868) : après compact, Claude « *gets lost and goes off on tangents* », se met à modifier du code sans rapport avec la session.
- [#13919](https://github.com/anthropics/claude-code/issues/13919) : contexte des Skills « *completely lost after auto-compaction* », l'agent « *repeating previously-fixed mistakes* » → « *massive productivity loss* ».
- [#36068](https://github.com/anthropics/claude-code/issues/36068) (« URGENT ») : « *drop user-stated intent and corrections from the context window* … *users must manually re-inject context, burning significant time and breaking flow* ».
- Aussi : [#3841](https://github.com/anthropics/claude-code/issues/3841), [#46695](https://github.com/anthropics/claude-code/issues/46695) (réglage de seuil demandé), [#33026](https://github.com/anthropics/claude-code/issues/33026), [#37914](https://github.com/anthropics/claude-code/issues/37914), [#66144](https://github.com/anthropics/claude-code/issues/66144), [#18314](https://github.com/anthropics/claude-code/issues/18314) — le thème génère des issues en continu depuis #868 (2025) jusqu'à #66144 (2026).
- L'outil communautaire *claunch* est né de ce constat : « *Claude Code has one major pain point: context doesn't persist between sessions* » ([DEV.to](https://dev.to/kaz123/how-i-solved-claude-codes-context-loss-problem-with-a-lightweight-session-manager-265d)).
- **→ NOVA WALL : 🟡 PARTIEL.** N'empêche pas la compaction, mais garde un **résumé sémantique persistant HORS de la session** (synergy.json : but/victoires/blocages extraits des transcripts) + flèches **inject/fusion** = filet pour réamorcer une session fraîche avec ce que l'ancienne savait. Manque : checkpoint automatique pré-compact + bouton « ressusciter cette session dans une neuve ».

### 🥈 P2 — Zéro visibilité de flotte : « le terminal n'a jamais été conçu comme un control plane »
Voir N agents d'un coup d'œil (qui tourne, qui fait quoi, qui a besoin de moi) n'existe pas nativement.
- **Citation exacte** (issue [#24537](https://github.com/anthropics/claude-code/issues/24537), fév. 2026, consolide **20+ issues**) : « *The Claude Code conversation view was designed for a human talking to one agent. It was never meant to be a control plane for 7 concurrent subagents across multiple sessions.* » Le demandeur liste ce qu'on **ne peut pas voir** : « *how many agents are running and what each is doing right now* », coût/contexte par agent, arbre des agents, progrès par agent, replay historique. Issues liées : [#7881](https://github.com/anthropics/claude-code/issues/7881) (19 👍), [#17425](https://github.com/anthropics/claude-code/issues/17425) (tree view), [#22625](https://github.com/anthropics/claude-code/issues/22625) (tokens par sous-agent).
- Preuve par l'écosystème (chaque outil = la douleur re-payée) : TmuxCC, claude-dashboard, [agents-observe](https://github.com/simple10/agents-observe), [AgentGlass](https://github.com/SirAllap/agentglass) (« *a real-time Mission-Control dashboard… across every provider* »), [claude-view](https://claudeview.ai/), [Mission Control](https://github.com/builderz-labs/mission-control), [Agent Deck](https://dev.to/asheshgoplani/how-to-manage-multiple-claude-code-gemini-and-codex-sessions-in-one-terminal-1dci) (« *terminal tabs becoming the bottleneck, not the coding itself* »), [Claude Squad](https://vibecodinghub.org/tools/claude-squad), [Crystal](https://munderdiffl.in/blog/best-claude-code-multi-agent-tools/), Conductor, Vibe Kanban, [Netra](https://medium.com/@nagarjuna.kanamarlapudi/claude-asked-permission-in-3-tabs-i-didnt-notice-for-3-hours-5d33e7357614)…
- Anthropic a fini par livrer **[agent view](https://code.claude.com/docs/en/agent-view)** (`claude agents`, research preview) — validation officielle de la douleur. Limites assumées de leur réponse : local à une machine, Claude-only, terminal-only.
- Même **Boris, créateur de Claude Code**, gère ça artisanalement : 5 onglets terminal **numérotés 1-5** + 5-10 sessions web + iOS = 15+ sessions parallèles ([breakdown DEV.to](https://dev.to/sivarampg/how-the-creator-of-claude-code-uses-claude-code-a-complete-breakdown-4f07)).
- **→ NOVA WALL : ✅ CŒUR DU PRODUIT** (mur de caméras : 51 sessions détectées / 10 live prouvé, badge modèle + fraîcheur + résumé sémantique par case, multi-moteurs Claude+Grok+GLM+local via tmux). Manques vs la demande #24537 : coût/tokens par case ❌, arbre des sous-agents ❌, replay ❌.

### 🥉 P3 — Babysitting : l'attention humaine est le goulot, pas la génération
- **Citations exactes** (HN « [Embracing the parallel coding agent lifestyle](https://news.ycombinator.com/item?id=45489884) ») :
  - simonw : « *AI-generated code needs to be reviewed, which means the natural bottleneck on all of this is how fast I can review the results* ».
  - munk-a : « *I can trust nothing and have to verify every assumption and that introduces a massive overhead* ».
  - grim_io : « *I'm not convinced there is any hope for a productive, long-term, burnout-free parallel agent workflow. Not while they need even the slightest amount of supervision/review* » (+ l'image de Sisyphe : « *the gods were kind enough to give him just one boulder, not 10 juggling balls* »).
  - WhyOhWhyQ : « *I can't seem to get myself to focus when one of these things is running. I transition into low effort mode.* »
- [Stop babysitting your AI coding agents](https://pushtoprod.substack.com/p/stop-babysitting-your-ai-coding-agents) : « *your productivity is constrained by your capacity for attention* » ; « *unless I keep sitting here, actively prompting and reviewing, no work gets done* » ; symptôme : 15-20 projets inachevés.
- [Aviator](https://www.aviator.co/blog/the-rise-of-coding-agent-orchestrators/) : « *You're spending as much time babysitting the AI as you would just writing the code yourself.* » ; gains réels « *closer to 20% versus vendor claims of 10x* ».
- **→ NOVA WALL : 🟡 PARTIEL.** Réduit le COÛT de supervision (état sémantique en un coup d'œil, blocages remontés au lieu de scroller 40 fenêtres) — mais ne fait pas la review de code (pas d'UI de diffs/PR). Positionnement honnête : NOVA WALL réduit le babysitting de PILOTAGE, pas le travail de review.

### P4 — Agents bloqués en silence sur un prompt de permission (« dead time »)
- **Citation exacte** ([Netra, Medium](https://medium.com/@nagarjuna.kanamarlapudi/claude-asked-permission-in-3-tabs-i-didnt-notice-for-3-hours-5d33e7357614)) : « *Three tabs. Three permission prompts. Three hours of dead time.* » et surtout : « ***I had 14 AI sessions open. Not one of them could tap me on the shoulder.*** »
- Issues : [#41639](https://github.com/anthropics/claude-code/issues/41639) (son sur permission prompt), [#51941](https://github.com/anthropics/claude-code/issues/51941) (alerte audio/visuelle, accessibilité), [#28765](https://github.com/anthropics/claude-code/issues/28765) (push en remote-control).
- Origine d'Omnara : partir déjeuner, revenir 20 min plus tard → « *the terminal sitting on a permission prompt that had been waiting the entire time* » ([happy.engineering/comparisons](https://happy.engineering/docs/comparisons/alternatives/)).
- Écosystème né de la douleur : [Claude Code Notifier](https://claudecodenotifier.com/), NotifyMe, hooks `Notification`, [guides « stop waiting for approval »](https://guides.happier.dev/how-to-fix-waiting-for-approval-in-claude-code-codex-and-opencode). Anthropic a ajouté les push mobiles ([docs](https://code.claude.com/docs/en/remote-control)).
- **→ NOVA WALL : 🟡 PARTIEL → ROADMAP.** La détection « session bloquée depuis 40 min » est dans le pitch, le pont Telegram existe ; mais les **alertes DAVID_ACTION_REQUIRED sont explicitement au « Reste »** du projet. À câbler : détection d'idle/permission + push + réponse one-tap (inject y/n).

### P5 — Conflits entre agents parallèles : « merge hell »
- Étude [AgenticFlict (arXiv 2604.03551)](https://arxiv.org/pdf/2604.03551) : sur **142K PRs d'agents**, **27,67 %** présentent des conflits de merge textuels — « *frequent and often substantial* ».
- HN : babyshake parle de « *merge hell* » ; réponse standard de la communauté = git worktrees ([DEV.to](https://dev.to/battyterm/git-worktrees-changed-how-i-run-parallel-ai-agents-39pm) : « *If two processes write to the same file at the same time, one overwrites the other* — broken code, lost work »). Diagnostic dédié : [Termdock](https://www.termdock.com/en/blog/git-worktree-conflicts-ai-agents).
- Issue [#34370](https://github.com/anthropics/claude-code/issues/34370) : isolation d'accès par worktree demandée. Même la réponse native (Agent Teams) admet : « *Task status can lag: teammates sometimes fail to mark tasks as completed, which blocks dependent tasks* » ([docs agent-teams](https://code.claude.com/docs/en/agent-teams)).
- **→ NOVA WALL : 🟡 ROADMAP.** La **détection de collision same-dossier** est explicitement listée au « Reste » (l'infra DreamNova a déjà bus coord + locks, pas encore packagés dans le mur). Pas de gestion worktree native.

### P6 — Coût/quota de flotte : « token burn » invisible et anxiogène
- L'outil le plus étoilé du domaine est un compteur de coûts : **[ccusage](https://github.com/ryoppippi/ccusage), 16 500 ⭐** (+ [Claude-Code-Usage-Monitor](https://github.com/Maciek-roboblog/Claude-Code-Usage-Monitor), [claude-usage](https://github.com/phuryn/claude-usage), Clusage…) — né du fait que le natif ne montre que la session courante.
- Anthropic ([agent view docs](https://code.claude.com/docs/en/agent-view)) : « *running ten agents in parallel uses quota roughly ten times as fast as running one* ».
- Gas Town (Yegge) : « *$100/hour burn rate* » ([paddo.dev](https://paddo.dev/blog/gastown-two-kinds-of-multi-agent/)). OpenHands : un agent en boucle a brûlé ~15 $ avant détection, extrapolé « *could exceed $100/hour* » à 10+ agents ([arXiv 2510.26585](https://arxiv.org/pdf/2510.26585)). L'auteur de Netra découvre **6 762 $** dépensés une fois le tracking construit.
- [Aviator](https://www.aviator.co/blog/the-rise-of-coding-agent-orchestrators/) : « *Cost economics: fleet operations become expensive* ».
- **→ NOVA WALL : ❌ PAS ENCORE.** Aucun compteur tokens/$ par case aujourd'hui. Feature quasi-gratuite à ajouter : lire les mêmes `.jsonl` que ccusage et afficher coût/fraîcheur par caméra + total mur.

### P7 — Contrôle à distance / mobile : « tethered to the desk »
- Toute une catégorie de produits est née de cette seule douleur : **[Omnara](https://apps.apple.com/us/app/omnara-claude-codex-mobile/id6748426727)** (9 $/mois, « *command center for your coding agents* »), **[Happy](https://happy.engineering/)** (open-source, E2E), et avant eux « *CodeRemote, YoloCode, Omnara, Claudia, Conductor, and Tonkotsu all tried* » ([happy.engineering](https://happy.engineering/docs/comparisons/alternatives/)).
- Anthropic a suivi : [Remote Control](https://code.claude.com/docs/en/remote-control) + push notifications mobiles (« *Start a heavy job in your terminal, walk away, and get notified on your phone* », [X/@v_shakthi](https://x.com/v_shakthi/status/2049292611291435051)).
- **→ NOVA WALL : ✅ OUI** (mur accessible web + pont Telegram, pilotage depuis le téléphone, lien protégé par jeton). Manque le raffinement Omnara : file d'**approbations** structurées one-tap (lié à P4).

### P8 — Le savoir reste prisonnier d'une session : erreurs re-payées, zéro apprentissage croisé
- **Citation exacte** (issue [#51735](https://github.com/anthropics/claude-code/issues/51735), avril 2026, **closed as not planned**) : « *corrections made in session N don't change behavior in session N+1, even when those corrections are explicitly documented in an archive that agents read and acknowledge* ». Exemples fournis : la même violation documentée re-commise 25 jours plus tard ; règle du CLAUDE.md ignorée dès le lendemain.
- Cette douleur recoupe P1 (le compact efface les leçons, #13919 « *repeating previously-fixed mistakes* ») et le scénario multi-sessions : une session re-paye le bug qu'une autre a déjà résolu.
- Seul embryon marché : Munder Difflin « *shared memory, inter-agent messaging* » ([comparatif](https://munderdiffl.in/blog/best-claude-code-multi-agent-tools/)), couplé à sa propre hive.
- **→ NOVA WALL : ✅ DIFFÉRENCIATEUR N°1.** Flèches **learn (🎓) / bridge (⇄) / inject (→)** + **fusion** + couche synergy (victoires réutilisables extraites des transcripts) = exactement ce que #51735 demande et qu'Anthropic a refusé de planifier. Personne d'autre ne relie les sessions entre elles.

### P9 — Sessions non portables entre machines / surfaces
- Issues : [#47926](https://github.com/anthropics/claude-code/issues/47926) « *Allow resuming Claude Code sessions across devices* », [#52052](https://github.com/anthropics/claude-code/issues/52052), [#31992](https://github.com/anthropics/claude-code/issues/31992) (« CLI-to-CLI handoff »).
- Douleur documentée : sessions indexées par CHEMIN de projet → copier `~/.claude` sur une autre machine casse le resume ([vincentschmalbach.com](https://www.vincentschmalbach.com/migrate-claude-code-sessions-to-a-new-computer/)) ; desktop app, web et extension VS Code ont chacun **leur propre historique** ([docs sessions](https://code.claude.com/docs/en/sessions)). Outil né de la douleur : [Claude Sync](https://dev.to/tawanorg/claude-sync-sync-your-claude-code-sessions-across-all-your-devices-simplified-49bl).
- **→ NOVA WALL : ❌ ROADMAP** (multi-machine/Dell explicitement au « Reste »). Le mur reste mono-Mac aujourd'hui.

### P10 — Confiance : agents qui « partent en vrille » sans qu'on le voie venir
- Devin : « *when a session goes wrong, you're watching an agent flail in an environment you don't control* » ([review](https://www.ayautomate.com/blog/devin-vs-claude-code)) ; taux de complétion « *single-digit to low-double-digit percent* » sur tâches complexes début 2025 (The Pragmatic Engineer, cité par [vibecoding.app](https://vibecoding.app/blog/devin-review)).
- Gas Town (Yegge, 20-30 Claude en `--dangerously-skip-permissions`) : « *the chaos is palpable* … *Bugs get fixed multiple times by different agents. Designs occasionally go missing* », tests en échec auto-mergés ([Welcome to Gas Town](https://steve-yegge.medium.com/welcome-to-gas-town-4f25ee16dd04), [LeoSimons](https://leosimons.com/2026/01/02/understanding-yegges-gas-town/)).
- OpenHands : terminaisons prématurées (« *frequently terminates prematurely after only one or two turns* », [arXiv 2506.14683](https://arxiv.org/pdf/2506.14683)). [Aviator](https://www.aviator.co/blog/the-rise-of-coding-agent-orchestrators/) : debugging multi-agents = « *whack-a-mole* », « *trust calibration* » non résolu. HN/voidhorse : « *you subconsciously lower the bar* ».
- **→ NOVA WALL : 🟡 PARTIEL.** On VOIT la dérive plus tôt (blocages + but affiché + capture tmux à la demande), mais pas de détection automatique de dérive ni de verdict qualité.

## 2) 5 FEATURES À FORT IMPACT — réclamées, offertes par personne
1. **Mémoire de flotte / transfert de savoir entre sessions** (inject → bridge → learn → fusion). Demande explicite [#51735](https://github.com/anthropics/claude-code/issues/51735) **fermée « not planned » par Anthropic** = boulevard. Aucun outil du marché ne fait circuler « ce que la session A a appris » vers B. *NOVA WALL l'a déjà — c'est l'angle d'attaque marketing.*
2. **Radar de collision en temps réel entre agents hétérogènes** (2 sessions, même dossier, fichiers qui se recouvrent → alerte/lock AVANT le merge). Le marché répond « discipline worktrees » ; personne ne détecte à chaud. Preuve du besoin : 27,67 % de PRs agentiques en conflit ([AgenticFlict](https://arxiv.org/pdf/2604.03551)).
3. **Inbox unifiée « needs input » multi-moteurs + réponse one-tap mobile** : UNE file de toutes les questions/permissions de TOUS les agents (Claude, Grok, GLM, local), avec détection heuristique « bloqué depuis X min ». Omnara/Happy = par-session et Claude/Codex-centriques ; agent view = local + Claude-only. La citation à afficher : « *I had 14 AI sessions open. Not one of them could tap me on the shoulder.* »
4. **Statut SÉMANTIQUE par agent** (but · réussites · blocages · client extraits du transcript) au lieu de logs bruts ou d'un kanban tenu à la main. C'est le « *what each is doing right now* » de [#24537](https://github.com/anthropics/claude-code/issues/24537) — les outils existants montrent le terminal brut (Claude Squad, Agent Deck) ou des statuts déclaratifs (Vibe Kanban).
5. **Fusion de deux sessions en une troisième** : le *fork* existe partout (agent view `/fork`, Agent Deck), le *merge* de contextes n'existe nulle part. Cas d'usage réclamé implicitement à chaque « une session a déjà résolu ce que l'autre re-paye ».

*(Bonus défensif : compteur coût/tokens par caméra — pas différenciant (ccusage), mais son absence est disqualifiante à terme, cf. P6.)*
